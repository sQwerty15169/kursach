# Отчет курсового проекта

## Тема
Полный цикл разработки и контейнеризации прикладного ПО с использованием экосистемы ИИ-ассистентов.

## 1) Сравнительный анализ ответов разных БЯМ на этапе прототипирования
Для старта проекта использовался единый промпт: "Сделай простое GUI-приложение менеджера заметок на Python".

Краткое сравнение:
- **DeepSeek**: быстро дал рабочий шаблон, но код был монолитным и без сохранения данных.
- **Qwen**: предложил более структурированный вариант, но с лишними зависимостями.
- **Kimi**: дал понятный и компактный базис, который было проще дорабатывать.

Вывод: в разработку принят подход с модульной структурой, CRUD-операциями и последующим расширением функционала.

## 2) Опыт взаимодействия с агентами и IDE-режимами
- На этапе проектирования полезен режим архитектурного планирования: выделение модели заметки, репозиториев и GUI-класса.
- Режим кодирования ускорил реализацию базового функционала.
- Режим отладки помог выявлять и исправлять логические ошибки в обработчиках интерфейса.

## 3) Реализованный программный продукт
Приложение "Менеджер заметок" выполнено на Python (`tkinter`) и поддерживает:
- создание/редактирование/удаление заметок;
- хранение заметок в JSON;
- поиск по заголовку и содержимому;
- переключение темы интерфейса (light/dark);
- логирование пользовательских действий.

## 4) Итеративные доработки
В процессе развития проекта добавлены расширенные функции:
1. Поиск по всем заметкам.
2. Переключение светлой и темной темы.
3. Логирование действий в файл `app.log` (дополнительная функция).

## 5) Git и Docker: выводы
- Git удобен для фиксации этапов разработки и возврата к стабильным версиям.
- Docker позволяет получить предсказуемое окружение и упростить переносимость.
- Основная сложность: запуск GUI внутри контейнера требует настройки X-сервера.

## 6) Состав отчетных материалов в проекте
- Исходный код: `main.py`
- Контекст и структура: `AGENTS.md`
- Контейнеризация: `Dockerfile`
